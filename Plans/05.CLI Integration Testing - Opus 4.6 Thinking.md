# CLI Integration Testing

Add a Jest-based integration test suite that validates OAuth token generation (live), then runs full autonomous pipelines for greenfield specs (`specs/new/`) and maintain-mode specs (`specs/existing/`).

---

## Context

The CLI (`src/cli.ts`) and conductor (`src/conductor/run.ts`) have no automated tests. Manual testing catches regressions late. The test suite will:

1. **Verify OAuth** — live call to `OAUTH_TOKEN_URL` returns a valid bearer token.
2. **Greenfield autonomous flow** — for each spec in `specs/new/`, call `runAutonomous()` and assert the final state is complete.
3. **Maintain autonomous flow** — for each spec in `specs/existing/`, call `runAutonomous()` in `maintain` mode against the previously generated project and assert the final state.

Tests run in sequence (OAuth → greenfield → maintain) because maintain tests depend on projects created by greenfield tests.

---

## Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Test framework | **Jest** with `ts-jest` | User preference; TypeScript support via `ts-jest` |
| OAuth test | **Live integration** | Validates real credentials from `.env` |
| Autonomous tests | **Programmatic** (`runAutonomous()`) | Direct assertions on `ProjectStateType`; no stdin piping |
| Timeout | **15 minutes** per autonomous test | LLM calls are slow; generous timeout avoids flaky failures |
| Test order | **Sequential** (Jest `--runInBand`) | Maintain tests depend on greenfield outputs |
| Spec discovery | **Dynamic** — glob `specs/new/*.txt` and `specs/existing/*.txt` | Adding a spec file automatically adds a test case |

---

## Step-by-Step Implementation

### Step 1: Install Jest & ts-jest

**File:** `package.json`

1. Add to `devDependencies`:
   ```json
   "jest": "^30.0.0",
   "ts-jest": "^29.4.0",
   "@types/jest": "^30.0.0"
   ```
2. Update `scripts`:
   ```json
   "test": "NODE_TLS_REJECT_UNAUTHORIZED=0 jest --runInBand --verbose",
   "test:oauth": "NODE_TLS_REJECT_UNAUTHORIZED=0 jest --runInBand --verbose --testPathPattern=oauth",
   "test:greenfield": "NODE_TLS_REJECT_UNAUTHORIZED=0 jest --runInBand --verbose --testPathPattern=greenfield",
   "test:maintain": "NODE_TLS_REJECT_UNAUTHORIZED=0 jest --runInBand --verbose --testPathPattern=maintain"
   ```
3. Run `npm install`.

### Step 2: Create Jest Config

**File:** `jest.config.ts`

```ts
import type { Config } from 'jest';

const config: Config = {
  preset: 'ts-jest',
  testEnvironment: 'node',
  roots: ['<rootDir>/tests'],
  testMatch: ['**/*.test.ts'],
  setupFiles: ['<rootDir>/tests/setup.ts'],
  testTimeout: 900_000, // 15 min default
};

export default config;
```

### Step 3: Create Test Setup File

**File:** `tests/setup.ts`

- Polyfill `globalThis.crypto` (same as `cli.ts`).
- Load `dotenv/config`.
- Validate that required env vars (`OAUTH_TOKEN_URL`, `OAUTH_CLIENT_ID`, `OAUTH_CLIENT_SECRET`, `LLM_BASE_URL`) are present; skip with a clear message if missing.

### Step 4: OAuth Token Test

**File:** `tests/oauth.test.ts`

```ts
describe('OAuth Token Generation', () => {
  it('should fetch a valid access token from the configured OAuth endpoint', async () => {
    const token = await getAccessToken();
    expect(token).toBeDefined();
    expect(typeof token).toBe('string');
    expect(token.length).toBeGreaterThan(0);
  });

  it('should return the same cached token on subsequent calls', async () => {
    const token1 = await getAccessToken();
    const token2 = await getAccessToken();
    expect(token1).toBe(token2);
  });
});
```

### Step 5: Greenfield Autonomous Flow Tests

**File:** `tests/greenfield.test.ts`

For each `.txt` file in `specs/new/`:

1. Derive `systemName` from filename (e.g., `simple-calculator.txt` → `SimpleCalculator`).
2. Call `runAutonomous({ systemName, requirementsDocPath, mode: 'autonomous', runType: 'greenfield' })`.
3. Assert on `finalState`:
   - `phase` === `'finalize'`
   - `workspacePath` exists on disk
   - `architecture` is not null
   - `userStories.length > 0`
   - `tasks.length > 0`
   - `assignments.length > 0`
   - `fileChanges.length > 0`
   - `testReports.length > 0`
   - `artifacts.length > 0`
   - `transcript.length > 0`

### Step 6: Maintain Autonomous Flow Tests

**File:** `tests/maintain.test.ts`

For each `.txt` file in `specs/existing/`:

1. Parse the spec to determine which existing project it targets (the spec text references "existing SimpleCalculator program" → look for the generated project from the greenfield run).
2. Resolve `existingProjectPath` to the generated project directory under `generated-projects/`.
3. Call `runAutonomous({ systemName, requirementsDocPath, mode: 'autonomous', runType: 'maintain', existingProjectPath })`.
4. Assert on `finalState`:
   - `phase` === `'finalize'`
   - `workspacePath` matches the existing project path
   - `codebaseAnalysis` is not null (maintain mode runs the codebase analyzer)
   - `fileChanges.length > 0` (new files were created/modified)
   - `testReports.length > 0`
   - `artifacts.length > 0`

### Step 7: Test Utilities

**File:** `tests/utils.ts`

Shared helpers:
- `discoverSpecs(dir: string)` — globs `*.txt` / `*.md` files and returns `{ systemName, filePath }[]`.
- `toSystemName(filename: string)` — converts `simple-calculator.txt` to `SimpleCalculator`.
- `findGeneratedProject(systemName: string)` — resolves path under `generated-projects/`.

## Running

```bash
# All tests (sequential: oauth → greenfield → maintain)
npm test

# Individual suites
npm run test:oauth
npm run test:greenfield
npm run test:maintain
```

Tests require a valid `.env` with OAuth credentials and LLM endpoint. They make real API calls and will take several minutes per autonomous run.
