# Split Base Schemas into Grouped Files

Refactor the monolithic `src/agents/_shared/base-schemas.ts` (346 lines, 13 schema groups) into individual files under `src/agents/_shared/schemas/`, then convert `base-schemas.ts` into a barrel re-export so **all existing imports continue to work unchanged**.

---

## Context

`base-schemas.ts` contains all Zod schemas for the domain entities — from RunInput to CodebaseAnalysis. The file is already organized with comment separators into logical groups. As the system grows (new agents, new schemas), this single file will become harder to navigate and maintain.

**Goal:** Split each comment-delimited group into its own file, keeping the existing `base-schemas.ts` path as a barrel re-export.

## Step-by-Step Implementation

### Step 1: Create the `schemas/` directory and individual files

Create `src/agents/_shared/schemas/` with 17 files, one per group above. Each file:
- Imports `z` from `'zod'`.
- Imports any cross-group dependencies (e.g. `approval.schema.ts` imports `PhaseNameSchema` from `./phase.schema`).
- Exports the schema(s) and inferred type(s).

**Cross-file dependencies:**
- `approval.schema.ts` → imports `PhaseNameSchema` from `./phase.schema`
- `transcript.schema.ts` → imports `PhaseNameSchema` from `./phase.schema`
- `codebase-analysis.schema.ts` → imports `CodebaseFileEntrySchema` (self-contained, uses inline)

### Step 2: Create a barrel `schemas/index.ts`

```ts
export * from './run-input.schema';
export * from './phase.schema';
export * from './architecture.schema';
export * from './tech-stack.schema';
export * from './epic.schema';
export * from './user-story.schema';
export * from './task.schema';
export * from './assignment.schema';
export * from './file-change.schema';
export * from './db-design.schema';
export * from './testing.schema';
export * from './bug.schema';
export * from './devops-plan.schema';
export * from './approval.schema';
export * from './artifact-ref.schema';
export * from './transcript.schema';
export * from './codebase-analysis.schema';
```

### Step 3: Convert `base-schemas.ts` to a barrel re-export

Replace the entire contents of `src/agents/_shared/base-schemas.ts` with:

```ts
/**
 * Barrel re-export of all domain schemas.
 *
 * Schemas have been split into individual files under ./schemas/ for readability.
 * This file preserves backward-compatibility so existing imports continue to work.
 */
export * from './schemas';
```

### Step 4: Verify

Run `npx tsc --noEmit` to confirm all imports resolve. No consumer file needs to change because `base-schemas.ts` still exports every symbol.

## Risks & Mitigations

- **Circular dependencies** → The only cross-file dep is `PhaseNameSchema` used by `approval` and `transcript`. These are one-directional, no risk.
- **Breaking imports** → Barrel re-export from `base-schemas.ts` ensures zero consumer changes needed.
